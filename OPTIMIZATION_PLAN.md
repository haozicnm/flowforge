## Phase 1: 稳定性与质量（1-2 周）

### 1.1 错误处理强化
- [ ] 统一错误类型：`FlowForgeError` 枚举
- [ ] 节点执行错误：包含节点 ID + 输入数据 + 错误上下文
- [ ] 超时机制：每个节点可配置超时（默认 30s）
- [ ] 重试策略：节点级重试配置（次数 + 间隔 + 指数退避）

### 1.2 测试覆盖
- [ ] 节点单元测试：每个节点至少 3 个测试（正常/边界/错误）
- [ ] 集成测试：工作流端到端执行
- [ ] API 测试：所有 HTTP 端点
- [ ] 目标：覆盖率 > 60%

### 1.3 日志与监控
- [ ] 结构化日志：tracing + JSON 输出
- [ ] 节点执行日志：输入/输出/耗时
- [ ] 工作流执行历史：SQLite 存储
- [ ] 错误告警：Webhook 通知

### 1.4 配置管理
- [ ] 配置文件：`flowforge.toml`
- [ ] 环境变量：`FLOWFORGE_*` 前缀
- [ ] 运行时配置：API 动态修改

## Phase 2: 节点系统增强（2-3 周）

### 2.1 核心节点补全
| 节点 | 功能 | 优先级 |
|------|------|--------|
| `email_send` | SMTP 发送邮件 | 高 |
| `email_read` | IMAP 读取邮件 | 高 |
| `ftp_upload` | FTP/SFTP 上传 | 中 |
| `ftp_download` | FTP/SFTP 下载 | 中 |
| `image_process` | 图片处理（裁剪/缩放/水印） | 中 |
| `pdf_extract` | PDF 文本提取 | 中 |
| `csv_parse` | CSV 解析 | 低 |
| `xml_parse` | XML 解析 | 低 |
| `base64_encode` | Base64 编解码 | 低 |
| `hash_generate` | 哈希生成 | 低 |

### 2.2 节点系统改进
- [x] 节点版本管理：`http@v2` 向后兼容
- [x] 节点验证：输入 schema 校验（JSON Schema）
- [x] 节点缓存：基础框架就绪
- [x] 节点并行：无依赖节点自动并行执行

### 2.3 变量系统增强
- [x] 全局变量：工作流级变量
- [ ] 环境变量注入：`${env.API_KEY}`
- [x] 表达式引擎：条件节点支持复杂表达式
- [x] 变量类型：string/number/boolean/array/object

### 2.4 插件系统
- [ ] 插件目录：`~/.flowforge/plugins/`
- [ ] 插件 manifest：`plugin.toml`
- [ ] 插件热加载：运行时加载/卸载
- [ ] 插件市场：社区插件分享
